Руководителю медицинской организации

ЛИС Akgun автоматизирует полный лабораторный цикл «направление → проба → анализ → верификация → выдача результата → аналитика», снижая операционные издержки и ускоряя оборот проб.

  • Приём заявок (order entry) из МИС / порталов / филиалов.
  • Управление пробами: штрих‑кодирование, трекинг перемещения, контроль сроков.
  • Интеграция с оборудованием (автоматический импорт результатов, исключение ручного ввода).
  • Правила авто‑валидации (рефлекс‑тесты, дельта‑контроль, критические значения).
  • Качество (QC): регистрация контролей, Westgard-правила (при включении модуля).
  • Формирование многоязычных отчётов для пациентов и врачей.
  • BI‑аналитика: TAT (turnaround time), загрузка приборов, частота отклонений.
  • Централизованный контроль филиалов и сравнение KPI.

ЛИС адаптирована для частных, государственных и сетевых лабораторий различного профиля:

  • Клиническая и общеклиническая лаборатория.
  • Биохимия, гематология, коагулология.
  • Микробиология (инкубационные циклы, посевы, антибиотикограмма).
  • Иммунология, серология, ИФА / ПЦР.
  • Цитология / гистология (при подключении модуля patho).
  • Токсикология (учёт цепочки хранения).

Гибкая конфигурация профилей тестов и референсных интервалов позволяет учитывать специфику каждого направления.

  • Практическая ориентация: формировалась с участием действующих лабораторий.
  • Модульная масштабируемость (добавление блоков без остановки ядра).
  • Глубокая интеграция с МИС / PACS / ERP / 1С / CRM.
  • Гибкая логика авто‑валидации (правила + дельта сравнение + критические значения).
  • Многофилиальность с консолидацией заказов и показателей качества.
  • Прозрачные аналитические панели (TAT, нагрузка, отклонения QC).
  • Мультиязычность интерфейса и отчётов.
  • 24/7 поддержка и эволюционное развитие по roadmap.

Мы применяем комплекс мер информационной безопасности:

  • Шифрование трафика (TLS), защита конфиденциальных полей в покое.
  • Ролевое + атрибутивное разграничение (филиал / роль / тестовый профиль).
  • Аудит действий: просмотр, редактирование, экспорт результатов.
  • Регулярные резервные копии, проверка восстановления, DR‑план.
  • Контроль целостности и мониторинг аномалий (массовые выгрузки).

Соответствие локальным регуляторным требованиям обеспечивается адаптацией форм и протоколов отчётности.

  • Разработана совместно с практикующими лабораториями.
  • Быстрый отклик 24/7 и проактивный мониторинг.
  • Постоянное развитие (roadmap + запросы клиентов).
  • Адаптивный интерфейс снижает время обучения персонала.
  • Масштабирование без деградации производительности.

Техническая поддержка и обучение входят в стоимость подписки; специализированные расширения оцениваются индивидуально.

Специалисту лаборатории

  • Штрих‑кодирование проб на этапе отбора (разные типы контейнеров).
  • Отслеживание статусов: зарегистрирована → в обработке → анализ → верификация → готово.
  • Контроль времени транспортировки и температурных условий (опциональные поля).
  • Предупреждения о просрочке хранения или истекших реагентах.
  • Лог цепочки действий для аудита (chain of custody).
  • Связка через ASTM / HL7 / специфичные драйверы поставщиков.
  • Двусторонний обмен: ЛИС отправляет заказы/панели, анализатор возвращает результаты.
  • Автоматическая маршрутизация повторных прогонов (re-runs) при флагах прибора.
  • Отметка критических и необычных значений (high/low/critical).
  • Минимизация ручного ввода — снижение риска транскрипционных ошибок.
  • Референсные интервалы по полу, возрасту, биоматериалу.
  • Дельта‑контроль (сравнение с предыдущим результатом пациента).
  • Рефлекс‑тестирование (автодобавление доп. анализа при условии).
  • Правила критических значений (мгновенные уведомления врачу).
  • Westgard‑правила для статистического контроля (при активном модуле QC).

Записи, не прошедшие авто‑валидацию, уходят в очередь ручной проверки.

  • Многоязычные шаблоны отчётов с логотипом учреждения.
  • Автоматическое объединение панелей в сводный документ.
  • Отметка референсов, патологических и критических отклонений цветом/значками.
  • Экспорт: PDF, отправка на e‑mail / портал пациента / МИС.
  • QR‑код (при необходимости) для верификации подлинности.
  • Приоритетизация проб (STAT / плановые).
  • Автоматическое распределение по доступным приборам.
  • Раннее выявление задержек (превышение целевых временных порогов).
  • Сокращение ручных операций (штрих‑коды, авто‑импорт результатов).
  • Авто‑валидация без участия специалиста для нормальных значений.

Интерфейс оптимизирован под частые действия (bulk операции, фильтры очередей). Обучение включает:

  • Стартовые сессии (онлайн / офлайн) по ролям.
  • Интерактивные подсказки и контекстная помощь.
  • Документация (FAQ, видео, чек‑листы).
  • Повторные тренинги при обновлениях или расширении штата.
  • Просмотр статусов проб и результатов с планшета (адаптивный веб).
  • Push / e‑mail уведомления о критических значениях.
  • Удалённое подтверждение результатов (при политике допуска).
  • Доступ к дашбордам TAT и QC для ответственных специалистов.

Системному администратору

  • Модульная слоистая архитектура: ядро (orders, samples), сервисы интеграции, отчётный модуль.
  • On‑premise / частное облако / гибрид (центральный ЦОД + филиалы).
  • Горизонтальное масштабирование критичных сервисов (очереди импорта, API‑шлюзы).
  • Кэширование справочников тестов / приборов / референсов.
  • HL7 v2.x (ORM, ORU, ADT) для заказов и результатов.
  • FHIR (Observation, Specimen, ServiceRequest, Patient).
  • ASTM / драйверы производителей анализаторов.
  • REST/JSON API для внешних порталов и интеграций.
  • CSV/XML импорт для пакетной миграции исторических данных.
  • Webhooks / message broker (Kafka/RabbitMQ) для событий (опционально).
  • RBAC + ограничение по филиалу/зоне лаборатории.
  • MFA (при включении), политики сложности паролей.
  • Шифрование конфиденциальных полей и токенизация (опционально).
  • Полный аудит действий: чтение, подтверждение, экспорт.
  • Аномалия‑детекция (массовые выборки, многократные логины).
  • Изолированные тестовые / staging среды для обновлений.
  • Очереди асинхронной обработки импорта результатов.
  • Шардирование/репликация БД при достижении порогов IOPS.
  • Кэширование часто запрашиваемых справочников и референсов.
  • APM‑мониторинг (latency, throughput, error rate).
  • Периодические нагрузочные тесты перед мажорными релизами.
  • Вертикальное + горизонтальное масштабирование контейнеризованных сервисов.
  • Полные и инкрементальные бэкапы БД + проверка восстановления (test restore).
  • Репликация критичных данных в резервный ЦОД.
  • Контроль целостности резервов (hash / checksum).
  • Документированный DR‑план с целями RPO/RTO.
  • Failover сценарии (manual / semi‑auto) для основных сервисов.
  • Staging среда для регрессионного тестирования перед продом.
  • Модули расширений используют публичные API — ядро не форкается.
  • Схема БД мигрирует через контролируемые скрипты (rollback возможно).
  • Release notes с перечнем изменений и требованиями.
  • Окна обновлений согласуются заранее (минимальный простой).
  • REST/JSON эндпоинты для заказов, образцов, статусов, результатов.
  • Webhook уведомления об изменении статусов (опционально включаются).
  • Документация (OpenAPI спецификация) и примеры запросов.
  • Контроль версий API (deprecated окна и совместимость).
  • Возможность разработки внутренних виджетов без изменения ядра.

Критические интеграции проходят совместное тестирование на пилотной среде.
